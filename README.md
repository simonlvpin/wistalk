# Wistalk

Wistalk 是一个浏览器端单页 Web 程序，用来把长篇材料拆解成可学习、可复盘、可迁移的结构化内容。

当前支持三类材料：

- 高层视角：适合 CEO 讲话、高层战略观点、管理分享，输出话题文章。
- 会议纪要：适合会议逐字稿、会议笔记、语音转文字结果，输出会议纪要分析，并按需提炼话题。
- 培训讲话：适合技术培训、业务培训、产品培训、方法论培训，按知识点/概念拆解培训内容，并按需提炼话题。

## 使用方式

1. 打开系统页面并登录。默认超级管理员是 `Simon.Lv@fanruan.com`，首次部署后请在本地初始化或修改管理员密码。
2. 在“材料上传”中上传 `.docx`、`.txt`、`.md` 文件。
3. 为每份材料选择资料来源和资料类型，例如 `CEO Marks / 高层视角`、`项目复盘会 / 会议纪要`、`技术培训 / 培训讲话`。
4. 点击“保存”，材料会进入未分析清单。
5. 勾选需要分析的材料，点击“执行分析”，系统会调用当前用户配置的大模型和对应资料类型的 SKILL。
6. 分析完成后，在“材料列表”或“话题列表”中学习结果。

## 当前能力

- 浏览器内解析 `.docx` 文件，不依赖后端服务。
- 使用浏览器 IndexedDB 保存用户、材料、配置和分析结果。
- 用户只能看到自己的材料；超级管理员可以查看所有用户内容。
- 支持申请试用、管理员生成初始密码、登录、退出、记住密码、修改密码和找回密码申请。
- 每位用户独立配置系统配色、字体大小、资料来源、资料类型和大模型参数。
- 每位用户的大模型 API Key 只保存在自己的浏览器本地数据中，不会进入 Git 仓库。
- 支持 DeepSeek 兼容接口：Base URL、模型名称和 API Key 由当前用户自行配置。
- 材料上传支持最多 10 个文件，单个文件不超过 50MB。
- 材料编号和话题编号唯一生成，便于长期引用。
- 材料列表支持按标题、标签、来源、类型和原文内容模糊搜索，并高亮关键词。
- 材料和话题都支持收藏，并可通过“只看收藏”快速筛选。
- 删除材料会联动删除相关话题；管理员可在回收站查看、恢复或彻底删除。
- 系统版本页记录每次更新日志；管理员可看到版本还原入口。

## SKILL 管理

- 不同资料类型拥有独立的 SKILL.md。
- 高层视角 SKILL 重点提炼“一个话题解决一个核心问题”的学习文章。
- 会议纪要 SKILL 重点输出会议信息、决策、行动项、议题讨论、协作态势、风险依赖和待定问题。
- 培训讲话 SKILL 重点按知识点/概念归类拆解，输出每个知识点的老师原文解释、讲解场景、注意问题、知识体系延伸、练习任务和整场培训的整体逻辑。
- 培训话题页使用培训学习模板，不沿用 CEO 高层讲话的问题化文章结构。
- 培训材料会按细颗粒知识点拆解话题，约一小时培训通常会提炼 10 个左右可学习话题。
- 培训分析页支持用户为每份材料、每个 SKILL 版本维护自己的富文本学习笔记。
- 如果会议纪要或培训讲话中没有清晰可提炼的话题，系统不会强行生成话题。
- SKILL 支持版本管理、差异日志、历史版本对比和材料刷新。
- 发布新 SKILL 后，可以批量筛选尚未使用最新版本的材料并重新执行分析。

## 数据与安全

- 本项目是静态前端应用，仓库不包含用户数据、浏览器 IndexedDB 数据、API Key、`.env`、`.db` 或 `.sqlite` 文件。
- 请勿把浏览器导出的数据库、私有配置、真实 API Key 或用户材料提交到 GitHub。
- GitHub Pages 仅托管程序代码，不托管你的本地数据。
